Details:

A userGroup is defined for administrators:



The members of this userGroup will have access to:

  • the 'Admin' panel (which you are currently visualizing) and,
  • to the service description editor in the 'Dossier' panel.

An organisationUnitGroupSet is defined to contain the list of services.



organisationUnitGroups are associated to the organisationUnitGroupSet as a list of services...
...and each of the organisationUnitGroups have a code.



To compile the information in the 'Dossiers' panel, the app reads:

  • the translations form the organisationUnitGroups names and descriptions and,
  • the code should be formatted as follows, for instance: *SomeText*_CD1 (CD1 would be used to associate to each service dataSets and indicatorGroups).

NB: As of dhis v2.25, an organisationUnitGroup should not be in more than one organisationUnitGroupSet, otherwise using it as a filter/dimension during the analysis will fail. As a consequence you might have to duplicate some of your organisationUnitGroups.

An attribute named 'serviceCode' is defined and associated with dataSets and indicatorGroups.

The serviceCode should correspond to, at least, one code of a service. They should be formatted as follows:

  • *AnotherText*_CD1 where CD1 is an example of code used in for a service.
  • One or more codes can be added in the case of dataSets or indicatorGroups used in multiple services: *AnotherText*_CD1_CD2_CD3

[Optional] An array of dataSets 'UID's to blacklist form the 'Search' panel.



[Optional] An array of indicatorGroups 'UID's to blacklist form the 'Search' panel.



The arrays should be formatted as follows: ["rD7MJ3LaakW","vnoUusJDY1Z","vCfO0z5igGT"].

About dataElements and indicators:

  • dataElements have to be associated to a dataSet and to a section to appear in the 'Dossiers' and the 'Search' panels.
    In the 'Search' panel, dataElements are displayed only from the 'Aggregated' domain.
    In the tables, the principal information used to display in columns is Name, Form Name and Description and their translations.

  • indicators have to be associated to a indicatorGroup to appear in the 'Dossiers' and the 'Search' panels.
    In the tables, the principal information used to display in columns is Name and Description and their translations.
    The field Description is splited in three parts as follows: *A_general_description* NUM:*A_description_of_the_numerator* DENOM:*A_description_of_the_denominator*. The key words NUM: and DENOM: should be respected to fill the related columns.
